# goap-optimized-simulation
GOAP AI optimization in C++ using Data-Oriented Design, bitmasking, caching, OpenMP, and tick staggering. Supports 262K agents @60 FPS and 524K @30 FPS on limited hardware, enabling scalable multi-layer AI systems.
#  High-Performance GOAP Simulation (C++)
> **Зорилгод суурилсан үйлдлийн төлөвлөлт (GOAP) алгоритмын санах ой болон тооцооллын оновчлол**

Энэхүү төсөл нь орчин үеийн видео тоглоом болон виртуал симуляцид олон мянган хиймэл оюун ухаант дүрийг (NPCs) нэгэн зэрэг удирдах үед үүсдэг CPU-ийн хэт ачааллын асуудлыг шийдвэрлэх зорилготой **шинэчилсэн архитектур бүхий эдийн засгийн симуляци** юм. Уламжлалт обьектод хандсан (OOP) хандлагаас татгалзаж, цэвэр өгөгдөл болон санах ойд чиглэсэн (Data-Oriented) инженерийн шийдлүүдийг C++ хэл дээр хэрэгжүүлсэн болно.

---

##  Онцлох шийдлүүд ба Архитектур

Энэхүү код нь энгийн нэгэн симуляци биш бөгөөд дараах 5 үндсэн оновчлолын тусламжтайгаар **524,288 агентыг зэрэг удирдах** хүчин чадалд хүрсэн:

1.  **Data-Oriented Design (SoA):** Агентуудын төлөвийг салангид обьектууд болгохын оронд санах ойд тасралтгүй үргэлжлэх дараалал хэлбэрээр байрлуулж, L1/L2 Cache-ийн ашиглалтыг дээд зэргээр нэмэгдүүлсэн.
2.  **Bitmasking (Битийн маск):** Агентын өлсгөлөн, мөнгө, бараа материал зэрэг олон хувьсагчийг 32-битийн ганц бүхэл тоонд (`uint32_t`) шахаж, санах ойн ашиггүй гарзыг устгасан.
3.  **Tick Staggering (Хугацааны хуваарилалт):** Хагас сая агент нэгэн зэрэг A* хайлт хийж системийг гацаахаас сэргийлж, хиймэл оюуны бодолтыг фреймүүдэд жигд хуваарилж (Amortization) ачааллыг тэнцвэржүүлсэн.
4.  **Generational Caching:** Давхардсан A* хайлтыг алгасахын тулд `ankerl::unordered_dense` ашиглан маш хурдан Hash-Map кэш санах ойг бүтээсэн. 
5.  **OpenMP Multithreading:** `std::atomic` ашиглан өгөгдлийн уралдаанаас (Data Race) сэргийлж, төлөвлөлтийн алгоритмыг олон цөм дээр зэрэгцүүлэн ажиллуулсан.

##  Шаардлагатай файлууд (Анхаарах зүйл!)

Энэхүү төсөлд уламжлалт C++ стандартын `std::unordered_map`-ийн оронд **хэд дахин илүү хурдтай, санах ойд хэмнэлттэй** `ankerl::unordered_dense` санг ашигласан болно. Тиймээс та кодоо ажиллуулахын тулд дараах 2 файлыг **заавал нэг хавтсанд хамт татаж авах шаардлагатай!**

 **Төслийн бүтэц:**
* `goap_simulationV13_1.cpp` *(Үндсэн симуляцийн эх код)*
* `ankerl/unordered_dense.h` *(C++ кэш системд зориулагдсан өндөр хурдны Hash-Map сан)*

##  Ашигласан технологиуд (Dependencies)

Кодыг амжилттай хөрвүүлж (Compile) ажиллуулахын тулд таны компьютерт дараах сангууд суусан байх шаардлагатай:
* **C++17** буюу түүнээс дээш хувилбар.
* **OpenMP** (Олон цөмийн зэрэгцээ тооцоолол хийхэд зориулагдсан).
* **Raylib** (2D график дүрслэл болон UI зурахад зориулагдсан үнэгүй сан).
